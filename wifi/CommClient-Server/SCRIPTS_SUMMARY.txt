SCRIPT FEATURES
================================================================================

install-service.ps1
  - Validates Python 3.10+
  - Creates virtual environment
  - Installs dependencies from requirements.txt
  - Auto-downloads NSSM if not found (via WinGet or direct download)
  - Registers Windows service "CommClientServer"
  - Sets startup type: Automatic
  - Configures restart on failure
  - Loads environment variables from .env file
  - Logs to: data/logs/service.log
  - Error handling: Comprehensive validation at each step

uninstall-service.ps1
  - Safe service removal with confirmation
  - Graceful shutdown before removal
  - NSSM cleanup
  - Verification of complete removal

start-server.ps1 & start-server.bat
  - Development mode launcher (no service)
  - Python version checking
  - Virtual environment creation
  - Dependency installation/updates
  - Database migration (alembic upgrade head)
  - Graceful Ctrl+C shutdown
  - Detailed startup info display

health-check.ps1
  - HTTP GET /api/health endpoint monitoring
  - Automatic service restart on failure
  - Log file auto-rotation (>10MB)
  - Diagnostic information gathering
  - Designed for Windows Task Scheduler
  - Exit codes: 0=healthy, 1=failed

backup-db.ps1
  - API-based backup (when server running + token)
  - Direct file backup (when server stopped)
  - ZIP compression support (7-Zip or PowerShell native)
  - Automatic rotation (keep N latest backups)
  - Database lock detection with retry logic
  - Integrity verification
  - Comprehensive logging

setup-firewall.ps1
  - Opens TCP 3000 (HTTP + Socket.IO)
  - Opens UDP 41234 (Discovery broadcast)
  - Opens UDP 40000-49999 (Mediasoup RTP streams)
  - Opens UDP 5353 (mDNS)
  - Per-profile configuration (Public/Private/Domain)
  - Enable/disable/remove operations
  - Connectivity verification

================================================================================
AUTOMATED TASK SETUP
================================================================================

HEALTH CHECK (Every 5 minutes with auto-restart):
  
  $trigger = New-ScheduledTaskTrigger -Daily -At "00:00" `
    -RepetitionInterval (New-TimeSpan -Minutes 5)
  $action = New-ScheduledTaskAction -Execute "powershell.exe" `
    -Argument '-ExecutionPolicy Bypass -File "C:\CommClient-Server\scripts\health-check.ps1" -AutoRestart'
  $principal = New-ScheduledTaskPrincipal -UserId "SYSTEM" -RunLevel Highest
  Register-ScheduledTask -TaskName "CommClientHealthCheck" `
    -Trigger $trigger -Action $action -Principal $principal

BACKUP (Daily at 2 AM):
  
  $trigger = New-ScheduledTaskTrigger -Daily -At "02:00"
  $action = New-ScheduledTaskAction -Execute "powershell.exe" `
    -Argument '-ExecutionPolicy Bypass -File "C:\CommClient-Server\scripts\backup-db.ps1" -Compress'
  $principal = New-ScheduledTaskPrincipal -UserId "SYSTEM" -RunLevel Highest
  Register-ScheduledTask -TaskName "CommClientBackup" `
    -Trigger $trigger -Action $action -Principal $principal

================================================================================
TROUBLESHOOTING
================================================================================

SERVICE WON'T START:
  1. Check Python: python --version (must be 3.10+)
  2. Check logs: Get-Content .\data\logs\service.log
  3. Check port: Test-NetConnection localhost -Port 3000
  4. Try manual: .\scripts\start-server.ps1

PORT ALREADY IN USE:
  1. Find process: Get-NetTCPConnection -LocalPort 3000
  2. Kill process: Stop-Process -Id <PID> -Force
  3. Or change port in .env and restart

DATABASE LOCKED:
  1. Stop service: Stop-Service CommClientServer -Force
  2. Wait 5 seconds
  3. Start service: Start-Service CommClientServer

FIREWALL ISSUES:
  1. Check rules: .\scripts\setup-firewall.ps1 -ListRules
  2. Re-enable: .\scripts\setup-firewall.ps1 -Enable
  3. Test port: Test-NetConnection <IP> -Port 3000

NSSM NOT FOUND:
  1. Check PATH: $env:Path -split ';'
  2. Manual download: https://nssm.cc/download
  3. Extract to C:\nssm or system PATH directory
