================================================================================
ALEMBIC MIGRATION SYSTEM - SETUP SUMMARY
CommClient-Server Database Schema Versioning
================================================================================

WHAT WAS CREATED
================================================================================

1. alembic.ini (project root)
   - Main Alembic configuration file
   - Points to migrations/ directory
   - SQLAlchemy URL placeholder (overridden by env.py)

2. migrations/env.py
   - Alembic environment configuration
   - Imports Base and all models for auto-discovery
   - Implements offline mode (SQL generation)
   - Implements online mode (live async migrations)
   - Supports both SQLite (aiosqlite) and PostgreSQL (asyncpg)
   - Reads database URL from app.core.config.get_settings()
   - Uses asyncio.run() for async engine lifecycle

3. migrations/script.py.mako
   - Jinja2 template for generated migration files
   - Standard Alembic format

4. migrations/versions/001_initial_schema.py
   - Initial migration that creates all 10 tables
   - Uses op.create_table() for each table
   - Includes all columns, types, constraints, and indexes
   - Includes downgrade() for full rollback

   Tables created:
   ✓ users (user accounts, profiles, status)
   ✓ user_sessions (multi-device JWT session tracking)
   ✓ contacts (buddy list relationships)
   ✓ channels (DM and group channels)
   ✓ channel_members (membership with roles)
   ✓ messages (text, file, system, replies)
   ✓ reactions (emoji reactions)
   ✓ files (file metadata and storage)
   ✓ call_logs (call history)
   ✓ message_receipts (delivery/read tracking)

5. MIGRATIONS.md
   - Comprehensive migration documentation
   - Usage examples and best practices
   - Troubleshooting guide
   - Environment variable reference

6. scripts/db_migrate.py
   - Convenient CLI wrapper for common tasks
   - Commands: status, upgrade, downgrade, reset, sql, history, etc.

================================================================================
COMMON COMMANDS
================================================================================

View migration status:
  alembic current
  alembic heads

Apply all pending migrations:
  alembic upgrade head

Downgrade one migration:
  alembic downgrade -1

Generate SQL for review:
  alembic upgrade head --sql

Create new migration:
  alembic revision --autogenerate -m "add column to users"

Reset database (dev only):
  alembic downgrade base
  alembic upgrade head

Using helper script:
  python scripts/db_migrate.py status
  python scripts/db_migrate.py upgrade
  python scripts/db_migrate.py history
  python scripts/db_migrate.py revision "your description"

================================================================================
ENVIRONMENT VARIABLES
================================================================================

DB_BACKEND
  - "sqlite" (default) - Use SQLite for development
  - "postgresql" - Use PostgreSQL for production

SQLITE_PATH (if DB_BACKEND=sqlite)
  - Default: ./data/commclient.db
  - Can be absolute or relative path
  - Directory created automatically

DATABASE_URL (if DB_BACKEND=postgresql)
  - PostgreSQL connection string
  - Format: postgresql+asyncpg://user:password@localhost/dbname
  - Required only for PostgreSQL backend

DEBUG
  - Controls SQLAlchemy engine echo (SQL logging)
