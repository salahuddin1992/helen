تشخيص الشبكة (Diagnostics) —

يكشف سبب مشاكل التوصيل (AP isolation، firewall، multicast filtering، subnet routing…) — كل نقطة يعرض السبب والحل.

ربط الخادم بالخارج (Connectivity)

يجرّب البرنامج المسارات بالترتيب: LAN مباشر → UPnP/NAT-PMP → Reverse Tunnel → Hole Punch → Relay. أول مسار ينجح يُستخدم.
إعداد Reverse Tunnel (rendezvous)
اتصال WebSocket صادر من هذا الخادم إلى rendezvous عام على الإنترنت. العملاء الخارجيون يصلون عبر عنوان rendezvous.
—
