1. Simplified UX Architecture

The redesigned CommClient eliminates all technical complexity from the user's view. The architecture follows a 3-level depth maximum — no user interaction requires more than 3 taps/clicks from the main screen.

Design Philosophy

Zero Configuration: The app auto-discovers the server via mDNS/UDP. The user never sees an IP address, port number, or connection setting. Login and Register are the only onboarding steps.

Progressive Disclosure: The main screen shows only 3 navigation tabs (Chats, People, Calls). All admin functions (RBAC, user management, server config) are hidden in Settings, visible only to admin-role users.

Forgiveness over Prevention: Destructive actions always require confirmation with clear consequences stated in plain language. Undo is preferred where possible.

Sidebar Design Rules

  • Width: 88px fixed. Icon (24px) + label (10px font) stacked vertically inside 60×60px button.
  • Active indicator: Light blue background (#EEF2FF) + blue text/icon (#4F6AFF).
  • Badge: Red circle (18px) in top-right corner of button. Only shown when count > 0.
  • Hover: Light gray background. Smooth 150ms transition.
  • No tooltips needed — every button has a visible text label below the icon.
  • Keyboard nav: Tab key cycles through sidebar buttons. Enter activates.

Error Display Rules

  • Toast position: Bottom-center, 440px max width, auto-dismiss after 5s
  • Icon: ⚠️ for warnings, ❌ for errors, ℹ️ for info
  • No error codes shown to user — codes logged to console for debugging
  • Retry button: Inline "Try again" for retryable actions (send message, make call)
  • Silent recovery: Token refresh, reconnection, and discovery retries happen automatically with no user notification unless they fail 3 times
