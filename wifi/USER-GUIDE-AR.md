## الإعداد الأول

### للمسؤول (تثبيت مرة واحدة)

أنت بحاجة إلى **جهاز سيرفر واحد** يعمل دائماً على شبكتك الداخلية. يمكن أن يكون:
- جهاز Windows عادي مع `Helen-Server-Setup-1.0.0.exe`
- جهاز Linux مع `helen-server-linux-1.0.0.tar.gz`
- جهاز macOS مع `helen-server-macos-1.0.0.tar.gz`
- أو NAS / Docker host

#### Windows
1. شغّل `Helen-Server-Setup-1.0.0.exe` كـ Administrator
2. اختر:
   - ☑ Helen-Server (مطلوب)
   - ☑ Install as Windows service (يبدأ تلقائياً)
   - ☑ Add Windows Firewall rules
3. الإعداد يحدث تلقائياً. لا تحتاج تعديل إعدادات.

#### Linux
```bash
sudo bash deploy/linux/scripts/install-server.sh \
    helen-server-linux-1.0.0.tar.gz
```
السكربت يضيف user + يثبّت service + يفتح firewall.

#### بعد التثبيت
- السيرفر يستمع على المنافذ: `3000` (HTTP)، `3443` (HTTPS)، `41234` (Discovery)
- اكتشف عنوان IP السيرفر: `ipconfig` (Windows) أو `ip addr` (Linux)

## تثبيت العميل

### Windows
1. شغّل `Helen Desktop Setup 1.0.0.exe`
2. التطبيق يفتح تلقائياً
3. أول مرة: يبحث عن السيرفر تلقائياً عبر LAN
4. لو لم يجده، اضغط "Manual Setup" وأدخل عنوان السيرفر:
   `http://192.168.1.5:3000`

### Linux
```bash
# Debian/Ubuntu
sudo dpkg -i commclient-desktop_1.0.0_amd64.deb

# باقي التوزيعات
chmod +x "Helen Desktop-1.0.0.AppImage"
./Helen\ Desktop-1.0.0.AppImage
```

### macOS
استخدم Web PWA (لا يوجد native build بعد):
1. افتح Safari
2. اذهب لـ `http://<server-ip>:3000`
3. File → Add to Dock

## تثبيت الموبايل

### Android
1. انقل `Helen-Mobile-1.0.0-release.apk` إلى الهاتف
2. اسمح بـ "Install from unknown sources" (Settings → Security)
3. اضغط على الملف لتثبيته
4. عند فتحه: اقبل صلاحيات الكاميرا، الميكروفون، الإشعارات

### iPhone / iPad (PWA)
1. افتح Safari
2. اذهب لـ `http://<server-ip>:3000`
3. اضغط زر **Share** → **Add to Home Screen**
4. الأيقونة تصير في الشاشة الرئيسية مثل تطبيق native

## الاستخدام اليومي

### تسجيل الدخول
- أول مستخدم في النظام يصير **owner** تلقائياً
- المستخدمون الجدد يحتاجون دعوة من owner أو admin

### إرسال رسالة
1. اختر قناة (channel) من العمود الأيسر
2. اكتب الرسالة في المربع السفلي
3. اضغط Enter للإرسال
4. لمرفق: اضغط الـ 📎 أو اسحب الملف للمحادثة

### اختصارات لوحة المفاتيح
| الاختصار | الوظيفة |
|---|---|
| `Enter` | إرسال |
| `Shift+Enter` | سطر جديد |
| `Ctrl+K` | البحث السريع |
| `Ctrl+/` | عرض كل الاختصارات |
| `Ctrl+,` | الإعدادات |
| `Esc` | إغلاق modal |

### البحث في الرسائل
- `Ctrl+F` داخل القناة
- `Ctrl+K` للبحث الشامل في كل القنوات
- يدعم العربية + UI dialect-aware

### إنشاء قناة جديدة
1. اضغط **+** بجانب "Channels"
2. اختر:
   - **Public** — كل المستخدمين يقدرون يدخلون
   - **Private** — بالدعوة فقط
   - **DM** — محادثة شخصية
3. أضف الأعضاء إن أردت

## المكالمات

### مكالمة صوتية 1:1
1. افتح محادثة DM
2. اضغط أيقونة 🎤
3. الطرف الآخر يرى دعوة، يقبل أو يرفض

### مكالمة فيديو
1. نفس الخطوات لكن اضغط 📹 بدلاً من 🎤
2. الكاميرا تشتغل تلقائياً (يطلب صلاحية مرة واحدة فقط)

### عناصر التحكم في المكالمة
- 🔇 كتم الميكروفون
- 📷 إخفاء الفيديو
- 🖥️ مشاركة الشاشة
- 🔴 تسجيل (لو مفعّل)
- ☎️ إنهاء المكالمة

### جودة الفيديو
الجودة تتكيف تلقائياً حسب سرعة الشبكة:
- **HD** (5 Mbps): 1080p
- **متوسطة** (2.5 Mbps): 720p
- **منخفضة** (800 Kbps): 480p
- **صوت فقط**: لو الشبكة سيئة

## مشاركة الشاشة

1. أثناء المكالمة، اضغط 🖥️
2. اختر:
   - **Entire screen** — كل الشاشة
   - **Application window** — نافذة محددة (Excel، Word، إلخ)
   - **Browser tab** — تبويب متصفح
3. الطرف الآخر يرى فوراً
4. لإيقاف المشاركة: اضغط 🖥️ مرة ثانية

### نصائح للأداء
- شارك **Application window** بدلاً من الشاشة كاملة (أسرع)
- أغلق الفيديو أثناء المشاركة لو الشبكة بطيئة
- إذا كنت تشارك شاشة عرض (Excel)، أغلق الفيديو الخاص بك

## مكالمات جماعية

### كم مشارك يدعم Helen؟
- **2** = peer-to-peer مباشر
- **3-6** = mesh (كل واحد يرى الكل)
- **7-50** = SFU (تتحوّل تلقائياً، الكل يرى آخر 12 متحدث)
- **51-200** = SFU كبير (آخر 8 متحدثين)
- **201-500** = SFU موزّع
- **501-2000** = Webinar mode (presenters فقط يتكلمون)

### إنشاء مكالمة جماعية
1. افتح قناة
2. اضغط 📹
3. كل أعضاء القناة يرون إشعار "join call"

### Webinar mode (>500 شخص)
عند تجاوز 500، النظام يتحوّل تلقائياً:
- 1-5 **presenters** يتكلمون ويظهرون فيديو
- الباقي **audience** يستقبلون فقط (لا ميكروفون، لا فيديو)
- audience يقدرون يكتبون في chat / Q&A

## حل المشاكل

### "لا يمكن الوصول للسيرفر"

#### الخطوة 1: تحقق من السيرفر
على جهاز السيرفر:
```bash
curl http://localhost:3000/api/health
# يجب يرجع: {"status":"ok",...}
```

#### الخطوة 2: تحقق من الـ ping
على جهاز العميل:
```cmd
ping <server-ip>
```
لو فشل: مشكلة شبكة، الجهازين مش بنفس الـ subnet.

#### الخطوة 3: شغّل diagnostic
```bash
python connection-diagnostic.py --host <server-ip> --port 3000
```
يفحص 16 احتمال ويعطي السبب الدقيق.

#### الخطوة 4: auto-fix
```cmd
# Run as Administrator
python helen-autofix.py --elevated
```
يصلح: firewall + Defender + hosts + clock + restart services.

### "المكالمة فيديو لا تعمل"

#### الكاميرا
- Windows: Settings → Privacy → Camera → سمح Helen
- macOS: System Preferences → Security & Privacy → Camera
- Android: التطبيق طلب صلاحية الكاميرا عند أول استخدام

#### الميكروفون
- نفس الخطوات لكن للميكروفون

#### "صوت يقطع"
- حاول إغلاق الفيديو
- تأكد من سرعة الشبكة (>1 Mbps لكل اتجاه)
- لو شبكة WiFi: اقترب من Router

### "مشاركة الشاشة لا تعمل"
- Windows 10/11: Settings → Privacy → Screen Recording → سمح Helen
- لو Picker فاضي: اعد تشغيل Helen

### "لا يمكنني تسجيل الدخول"
- تأكد من اسم المستخدم وكلمة السر
- لو نسيت السر: اطلب من admin يعمل reset
- تأكد من وقت الجهاز صحيح (skew >5 ثوان يكسر JWT)

### "إشعارات لا تصل"
- Helen يستخدم WebSocket persistent (لا يحتاج FCM/APNs)
- على Android: تأكد التطبيق ليس في "battery saver" المقيّد
- على iPhone: لازم تكون فاتح Safari (PWA limitation)

## الأمان

### تشفير الرسائل (E2EE)
- الرسائل المشفّرة بين العملاء (X3DH + Double Ratchet مثل Signal)
- حتى مسؤول السيرفر لا يقدر يقرأها
- تظهر 🔒 على الرسائل المشفّرة

### كلمة السر
- 8 حروف على الأقل
- مزيج أرقام + حروف + رموز
- لا تستخدمها في أي موقع آخر
- مدير كلمات سر داخلي مدعوم (Vault)

### فقدان جهازك
1. اطلب من admin: "logout from device X"
2. غيّر كلمة السر فوراً
3. كل sessions الأخرى تطفي تلقائياً

### Vault — الأسرار الحساسة
- افتح من القائمة الجانبية → Vault
- فقط أنت تقدر تفتحها بـ Vault password (ليس كلمة سر Helen)
- ملفات Vault مشفّرة بمفتاح مشتق من passwordك

## أسئلة شائعة

### هل Helen يحتاج إنترنت؟
**لا.** Helen يعمل 100% داخل شبكتك المحلية (LAN/WiFi/Fiber الداخلي). أيّ ميزة لا تحتاج إنترنت.

### هل بياناتي تذهب لأي شركة؟
**لا.** كل البيانات على سيرفرك الداخلي. لا cloud, لا Google, لا Facebook, لا Microsoft.

### كم يستغرق التثبيت؟
- السيرفر: ~5 دقائق
- العميل: ~30 ثانية
- Mobile: ~1 دقيقة (تثبيت APK + صلاحيات)

### كم مستخدم يدعم النظام؟
- **سيرفر واحد**: حتى 5,000 مستخدم نشط
- **multiple servers + router**: غير محدود نظرياً (اختُبر 1 مليون router)

### هل يمكنني استيراد محادثاتي من Slack؟
نعم. اطلب من admin يستخدم `tools/import_slack.py` مع Slack workspace export.

### كيف أحدّث Helen؟
- التحديث **لا يتم تلقائياً** (آمن)
- admin يضع الإصدار الجديد على update server
- المستخدمون يرون "Update Available" → يضغطون "Install"

### هل أحتاج حساب على Helen؟
نعم. أوّل مستخدم في النظام يصير admin، ويدعو الآخرين.
